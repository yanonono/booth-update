Linear Projectile 紹介文

Requires FinalIK 1.71 A linear projectile with positional effects on collision. Message lindesu#0660 for support https://cdn.discordapp.com/attachments/524149158479921152/551755421942284298/unknown.png install it at root, move the linear projectile ik script to root as well position and rotate the linear projectile object so the y direction is pointing where you need. add a rigidbody to the fixed joint component so that the linear projectile object tracks something use the override to test with default effects in-game, then start replacing the effects for the most seamless projectile effects, do all of your visual effects as particles under the firing effects using the velocity particle mirror and have the traveling and detonation effects just contain audio. otherwise you can put game objects in the traveling effects and detonation effects there is a safety lock animator on the main object. when it's on, it will prevent the system from firing. keep the main object on because of the joint, and animate a default on safety lock to off with behavior.enabled 0 when you pull out your gun, or something you need final ik. you dont need execution order or anything. just have final ik imported. message me if you need help on this step. have an sdk imported for the humanoid dependency careful when simulating it in play mode, the reset doesn't work how you would intuitively expect and its hard to simulate without a gesture enforcing defaults. i recommend just confirming it fires with the start animator and then ending play mode. make sure its far enough away from your avatar pill or it will misfire in game the only joint to care about is the main object joint, the rest uses ik. i haven't run deeper ik experiments. you can probably ik the linear projectile object to something, and armature parenting might work if you can fix all the world particles with ik follower :shrug: rigidbody is what im supporting right now, as limbik wont track limbs in fbt and it works well
